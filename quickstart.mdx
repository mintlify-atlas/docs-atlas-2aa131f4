## Prerequisites

Before you begin, you need one of the following:

- **Docker** (recommended for the fastest setup)
- **PostgreSQL 11–18** with AGE installed — see the [Installation guide](/installation)

## Run AGE with Docker

<Steps>
  <Step title="Pull the Docker image">
    Pull the official Apache AGE image from Docker Hub:

    ```bash
    docker pull apache/age
    ```
  </Step>

  <Step title="Start the container">
    Create and start an AGE container with the following command:

    ```bash
    docker run \
        --name age  \
        -p 5455:5432 \
        -e POSTGRES_USER=postgresUser \
        -e POSTGRES_PASSWORD=postgresPW \
        -e POSTGRES_DB=postgresDB \
        -d \
        apache/age
    ```

    This maps port `5455` on your host to PostgreSQL's default port `5432` inside the container.
  </Step>

  <Step title="Connect to PostgreSQL">
    Open a `psql` session inside the running container:

    <Tabs>
      <Tab title="Docker exec">
        ```bash
        docker exec -it age psql -d postgresDB -U postgresUser
        ```
      </Tab>
      <Tab title="psql (external client)">
        If you have `psql` installed locally, connect using the mapped port:

        ```bash
        psql -h localhost -p 5455 -d postgresDB -U postgresUser
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Load the AGE extension">
    Run the following three statements at the start of every AGE session:

    ```sql
    CREATE EXTENSION age;
    LOAD 'age';
    SET search_path = ag_catalog, "$user", public;
    ```

    <Note>
      `CREATE EXTENSION age` only needs to run once per database. `LOAD 'age'` and `SET search_path` must be run for each new connection.
    </Note>
  </Step>

  <Step title="Create a graph">
    Use the `create_graph` function in the `ag_catalog` namespace to create a new named graph:

    ```sql
    SELECT create_graph('graph_name');
    ```
  </Step>

  <Step title="Create vertices">
    Use the Cypher `CREATE` clause to add vertices with a label and properties:

    ```sql
    SELECT *
    FROM cypher('graph_name', $$
        CREATE (:label {property:"Node A"})
    $$) as (v agtype);
    ```

    ```sql
    SELECT *
    FROM cypher('graph_name', $$
        CREATE (:label {property:"Node B"})
    $$) as (v agtype);
    ```
  </Step>

  <Step title="Create an edge">
    Use `MATCH` to find your vertices, then `CREATE` to add an edge between them:

    ```sql
    SELECT *
    FROM cypher('graph_name', $$
        MATCH (a:label), (b:label)
        WHERE a.property = 'Node A' AND b.property = 'Node B'
        CREATE (a)-[e:RELTYPE {property:a.property + '<->' + b.property}]->(b)
        RETURN e
    $$) as (e agtype);
    ```
  </Step>

  <Step title="Query the graph">
    Use `MATCH` to retrieve connected vertices and edges:

    ```sql
    SELECT * FROM cypher('graph_name', $$
        MATCH (V)-[R]-(V2)
        RETURN V,R,V2
    $$) as (V agtype, R agtype, V2 agtype);
    ```

    You should see both vertices and the edge you created, returned as `agtype` values.
  </Step>
</Steps>
