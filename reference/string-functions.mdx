---
title: "String functions"
description: "Cypher string functions implemented in ag_catalog for transforming and extracting text from agtype values."
---

String functions accept `agtype` string arguments and return `agtype` values. All SQL implementations are in the `ag_catalog` schema, use `VARIADIC "any"` signatures, and are declared `IMMUTABLE PARALLEL SAFE`.

All functions return `NULL` when passed a `NULL` input.

---

## Case and whitespace

### toUpper

```sql
ag_catalog.age_toupper(VARIADIC "any") → agtype
```

Converts a string to upper case.

| Cypher name | SQL name                        |
| ----------- | ------------------------------- |
| `toUpper()` | `age_toupper(VARIADIC "any")`   |

```sql
SELECT * FROM ag_catalog.cypher('my_graph', $$
    RETURN toUpper('hello world')
$$) AS (result agtype);
-- "HELLO WORLD"
```

## Substring extraction

### left

```sql
ag_catalog.age_left(VARIADIC "any") → agtype
```

Returns the first `count` characters of a string.

| Cypher name | SQL name                      |
| ----------- | ----------------------------- |
| `left()`    | `age_left(VARIADIC "any")`    |

<Tip>
  Pass the string as the first argument and the character count as the second.
</Tip>

```sql
SELECT * FROM ag_catalog.cypher('my_graph', $$
    RETURN left('Apache AGE', 6)
$$) AS (result agtype);
-- "Apache"
```

### substring

```sql
ag_catalog.age_substring(VARIADIC "any") → agtype
```

Returns a portion of a string starting at a zero-based index with an optional length.

| Cypher name   | SQL name                          |
| ------------- | --------------------------------- |
| `substring()` | `age_substring(VARIADIC "any")`   |

```sql
-- substring(string, start)
SELECT * FROM ag_catalog.cypher('my_graph', $$
    RETURN substring('Apache AGE', 7)
$$) AS (result agtype);
-- "AGE"

-- substring(string, start, length)
SELECT * FROM ag_catalog.cypher('my_graph', $$
    RETURN substring('Apache AGE', 0, 6)
$$) AS (result agtype);
-- "Apache"
```

## Usage with vertex properties

String functions compose naturally with property access inside a Cypher query:

```sql
SELECT * FROM ag_catalog.cypher('my_graph', $$
    MATCH (n:Person)
    RETURN n.name, toUpper(n.name), trim(n.email)
$$) AS (name agtype, upper_name agtype, email agtype);
```
