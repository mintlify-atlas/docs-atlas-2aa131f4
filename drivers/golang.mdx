---
title: "Go driver"
description: "Connect to Apache AGE from Go using the database/sql API or the higher-level Age Wrapper."
---

The Go driver provides agtype parsing and Cypher query support for Apache AGE on top of the standard [`database/sql`](https://golang.org/pkg/database/sql/) package. It uses ANTLR 4 to parse agtype results and exposes two usage patterns: a low-level SQL API and a higher-level Age Wrapper.

## Prerequisites

- Go 1.18 or later
- Java 11 or later (required to generate the ANTLR parser at build time)
- ANTLR 4.11.1 or later

## Usage pattern 1: database/sql API

Use the standard `database/sql` `*sql.Tx` API together with the `age.ExecCypher` function. This pattern gives you explicit transaction control.

```go
package main

import (
    "database/sql"
    "fmt"

    "github.com/apache/age/drivers/golang/age"
)

func doWithSqlAPI(dsn string, graphName string) {

    // Connect to PostgreSQL
    db, err := sql.Open("postgres", dsn)
    if err != nil {
        panic(err)
    }

    // Confirm graph exists (creates it if not present)
    _, err = age.GetReady(db, graphName)
    if err != nil {
        panic(err)
    }

    // Begin a transaction for write operations
    tx, err := db.Begin()
    if err != nil {
        panic(err)
    }

    // Create vertices with Cypher
    _, err = age.ExecCypher(tx, graphName, 0, "CREATE (n:Person {name: '%s', weight:%f})", "Joe", 67.3)
    if err != nil {
        panic(err)
    }

    _, err = age.ExecCypher(tx, graphName, 0, "CREATE (n:Person {name: '%s', age: %d})", "Jack", 30)
    if err != nil {
        panic(err)
    }

    tx.Commit()

    // Begin a read transaction
    tx, err = db.Begin()
    if err != nil {
        panic(err)
    }

    // Query vertices
    cypherCursor, err := age.ExecCypher(tx, graphName, 1, "MATCH (n:Person) RETURN n")
    if err != nil {
        panic(err)
    }

    // Unmarshal results row by row
    for cypherCursor.Next() {
        row, err := cypherCursor.GetRow()
        if err != nil {
            panic(err)
        }
        vertex := row[0].(*age.Vertex)
        fmt.Println(vertex.Id(), vertex.Label(), vertex.Props())
    }
}
```

### Querying paths

```go
// Create edges
_, err = age.ExecCypher(tx, graphName, 0,
    "MATCH (a:Person), (b:Person) WHERE a.name='%s' AND b.name='%s' CREATE (a)-[r:workWith {weight: %d}]->(b)",
    "Jack", "Joe", 3)
if err != nil {
    panic(err)
}
tx.Commit()

tx, err = db.Begin()
if err != nil {
    panic(err)
}

// Query paths
cypherCursor, err = age.ExecCypher(tx, graphName, 1, "MATCH p=()-[:workWith]-() RETURN p")
if err != nil {
    panic(err)
}

for cypherCursor.Next() {
    row, err := cypherCursor.GetRow()
    if err != nil {
        panic(err)
    }
    path := row[0].(*age.Path)
    vertexStart := path.GetAsVertex(0)
    edge        := path.GetAsEdge(1)
    vertexEnd   := path.GetAsVertex(2)
    fmt.Println(vertexStart, edge, vertexEnd)
}
```

### Returning multiple columns

The third argument to `ExecCypher` is the expected column count. Pass the number of values your `RETURN` clause yields:

```go
// RETURN a, l, b — three columns
cursor, err := age.ExecCypher(tx, graphName, 3,
    "MATCH (a:Person)-[l:workWith]-(b:Person) RETURN a, l, b")
if err != nil {
    panic(err)
}

count := 0
for cursor.Next() {
    row, err := cursor.GetRow()
    if err != nil {
        panic(err)
    }
    count++
    v1   := row[0].(*age.Vertex)
    edge := row[1].(*age.Edge)
    v2   := row[2].(*age.Vertex)
    fmt.Println("ROW", count, ">>", v1, edge, v2)
}
```

## Usage pattern 2: Age Wrapper

The Age Wrapper provides a higher-level API that manages the connection and graph context for you.

```go
package main

import (
    "fmt"

    "github.com/apache/age/drivers/golang/age"
)

func doWithAgeWrapper(dsn string, graphName string) {

    // Connect and bind to a graph
    ag, err := age.ConnectAge(graphName, dsn)
    if err != nil {
        panic(err)
    }

    tx, err := ag.Begin()
    if err != nil {
        panic(err)
    }

    // Create vertices
    _, err = tx.ExecCypher(0, "CREATE (n:Person {name: '%s'})", "Joe")
    if err != nil {
        panic(err)
    }

    _, err = tx.ExecCypher(0, "CREATE (n:Person {name: '%s', age: %d})", "Smith", 10)
    if err != nil {
        panic(err)
    }

    _, err = tx.ExecCypher(0, "CREATE (n:Person {name: '%s', weight:%f})", "Jack", 70.3)
    if err != nil {
        panic(err)
    }

    tx.Commit()

    // Query vertices
    tx, err = ag.Begin()
    if err != nil {
        panic(err)
    }

    cursor, err := tx.ExecCypher(1, "MATCH (n:Person) RETURN n")
    if err != nil {
        panic(err)
    }

    count := 0
    for cursor.Next() {
        entities, err := cursor.GetRow()
        if err != nil {
            panic(err)
        }
        count++
        vertex := entities[0].(*age.Vertex)
        fmt.Println(count, "]", vertex.Id(), vertex.Label(), vertex.Props())
    }

    fmt.Println("Vertex Count:", count)
}
```


## Connection DSN

The DSN format follows the `lib/pq` convention:

```go
var dsn string = "host=127.0.0.1 port=5432 dbname=postgres user=postgres password=agens sslmode=disable"
var graphName string = "testGraph"
```

## Agtype types

| AGE type | Go type | Key methods |
|----------|---------|-------------|
| Vertex | `*age.Vertex` | `.Id()`, `.Label()`, `.Props()` |
| Edge | `*age.Edge` | `.Id()`, `.Label()`, `.Props()` |
| Path | `*age.Path` | `.GetAsVertex(i)`, `.GetAsEdge(i)` |

## Testing

Before running the tests:

1. Install the driver following the steps above.
2. Ensure your PostgreSQL database has AGE installed.
3. Create a graph named `testGraph` in that database.
4. Update the DSN in `age/drivers/golang/age/age_test.go`.

```bash
cd age/drivers/golang/age
go test . -v
```
