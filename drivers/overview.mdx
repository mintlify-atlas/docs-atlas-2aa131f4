title: "Language drivers overview"
description: "Connect to Apache AGE from Python, Go, Java, and Node.js using official language drivers."
---

Apache AGE provides official drivers for four languages. Each driver handles the agtype wire format — unmarshalling AGE's native `agtype` results into language-native `Vertex`, `Edge`, and `Path` types so you work with structured objects rather than raw strings.

<Note>
All drivers require a running PostgreSQL instance with the AGE extension installed and loaded. Run the following in `psql` before connecting from any driver:

```sql
CREATE EXTENSION age;
LOAD 'age';
SET search_path = ag_catalog, "$user", public;
```
</Note>

## Official drivers

<CardGroup cols={2}>
  <Card title="Python" icon="python" href="/drivers/python">

  </Card>
  <Card title="Go" icon="golang" href="/drivers/golang">
    Built on `database/sql`. Provides both a low-level SQL API and a higher-level Age Wrapper.
  </Card>
  <Card title="Java (JDBC)" icon="java" href="/drivers/java">

  </Card>
  <Card title="Node.js" icon="node-js" href="/drivers/nodejs">

## Agtype parsing

Apache AGE returns query results in its own binary format called **agtype**. Each driver includes an ANTLR 4-based parser that converts agtype values into first-class objects:

| AGE type | Python | Go | Java | Node.js |
|----------|--------|----|------|---------|
| Vertex | `Vertex` | `*age.Vertex` | `Agtype` (map) | parsed object |
| Edge | `Edge` | `*age.Edge` | `Agtype` (map) | parsed object |
| Path | `Path` | `*age.Path` | `Agtype` (map) | parsed object |

You access vertex and edge properties through the returned objects — no manual JSON parsing required.

## Common prerequisites

All drivers share these requirements:

- A running PostgreSQL instance (version 11 or later)
- The [Apache AGE extension](https://github.com/apache/age) installed in PostgreSQL
- The AGE extension loaded in each session (`LOAD 'age'`)
- The `ag_catalog` schema on the search path
